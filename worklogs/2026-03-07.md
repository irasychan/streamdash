## Architecture Decisions

- **`moderatedUserIds` is keyed by `author.id`** (not message ID) so all messages from a banned/timed-out user dim at once, matching Twitch's native behavior.
- **`moderatedUserIds` is ephemeral** (in-memory, lost on page refresh). Persisting it belongs in the SQLite phase (3.1).
- **`isModerated` styling defers to `isHidden`** -- if a message is both hidden and from a moderated user, the `isHidden` grey style takes precedence (via `!isHidden` guard).

### Message Highlighting
**Priority:** High

Implemented `@mention` and keyword inline highlighting in the message render pipeline.

- `highlightText()` utility added to emoteRenderer — builds a combined regex, splits text segments, wraps matches in styled spans
- `@mention` → `text-primary font-semibold bg-primary/10 rounded px-0.5` (gold)
- keyword → `text-accent font-semibold bg-accent/10 rounded px-0.5` (cyan)
- Emote images, usernames, and badges are unaffected
- Preferences (`highlightMentions`, `highlightKeywords`) were already defined in types/defaults — only rendering needed implementing
- Verified via Playwright: keywords "stream" and "everyone" highlighted correctly in demo messages

**Files modified:**
- `features/chat/utils/emoteRenderer.tsx` — `highlightText()` utility + integrated into text-segment step
- `features/chat/components/ChatMessage.tsx` — extracts highlight prefs from `chatPrefs`, passes to `renderMessageWithEmotes`

## Architecture Decisions

- Highlighting integrated into the emote renderer's text-segment step (not post-processing React nodes) — ensures highlights never apply to emote images or badges
- H shortcut uses a `setSelectedMessageId` updater callback to read current hidden state without extra closure complexity
